# 下一次纯净重装操作清单

版本：3.0.0-rc3 草稿  
更新：2026-07-21  
适用：常见 VPS、Debian 12/13、VMess + WebSocket + TLS、Cloudflare 官方 WARP、macOS/iPhone Shadowrocket

> 这仍是尚未完成全新 VPS 端到端实测的草稿。当前稳定服务器不需要为了测试而重装。真正重装前，只需让 Codex 再确认一次本页安装入口是否仍是准备使用的版本。

## 先看这一段

正常重装只有一条主线：

1. 在自己的电脑生成 SSH 密钥，只把公钥交给 VPS 供应商。
2. 选择 Debian 13，记录服务器公网 IPv4，并允许入站 TCP 22、80、443。
3. 把一个域名的 A 记录改为该 IPv4，保持 DNS only，不发布同名 AAAA。
4. 用自己的 SSH 私钥登录服务器，整行复制一次安装命令。
5. 输入域名、证书通知邮箱和一次大写 `YES`，等待自动验收。
6. 只有最终显示 `Result: PASS` 后，脚本才显示客户端链接和二维码。

不再要求两个 SSH 窗口，也不要求会用 Serial/VNC Console。脚本会在 WARP 验证前停止 V2Ray，并在路由切换时启用本机定时回退；WARP 未完成时不会交付可用节点。

## 一、准备自己的 SSH 密钥

一套密钥可以继续用于以后更换 VPS 供应商。私钥只保存在自己的设备和加密备份中。

### Windows

在 PowerShell 中执行：

```powershell
New-Item -ItemType Directory -Force "$HOME\.ssh" | Out-Null
ssh-keygen -t ed25519 -a 100 -f "$HOME\.ssh\vps_senyz_ed25519" -C "senyz-vps"
icacls "$HOME\.ssh\vps_senyz_ed25519" /inheritance:r
icacls "$HOME\.ssh\vps_senyz_ed25519" /grant:r "$($env:USERNAME):(R)"
Get-Content "$HOME\.ssh\vps_senyz_ed25519.pub"
```

`ssh-keygen` 会要求输入两次密钥口令。最后一条显示的是可以上传的公钥。没有 `.pub` 后缀的是私钥，不上传到 VPS、GitHub、聊天或网盘公开目录。

### macOS

在“终端”中执行：

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
ssh-keygen -t ed25519 -a 100 -f ~/.ssh/vps_senyz_ed25519 -C senyz-vps
chmod 600 ~/.ssh/vps_senyz_ed25519
cat ~/.ssh/vps_senyz_ed25519.pub
```

把私钥做一份加密备份。原来的 `id_rsa.pem` 暂时保留，等新服务器和客户端都稳定后再决定是否归档。

## 二、选择和重装 VPS

脚本不依赖 DMIT，供应商只需满足：

- 提供独立公网 IPv4；
- 可以安装 Debian 12 或 13，优先 Debian 13；
- 可以把自己的 SSH 公钥写入 root 账户；
- 使用 systemd，根分区至少有 2 GiB 可用空间；
- 供应商防火墙或 Security Group 能允许入站 TCP 22、80、443。

在供应商控制面板上传上一节的整行 `.pub` 公钥，然后重装系统并选择该公钥。不要把私钥粘贴到网页。

如果供应商不支持自定义公钥、只允许下载它生成的私钥，或默认不能用 root 公钥登录，先暂停并把对应页面截图交给 Codex；不要临时改成密码登录来绕过。

DMIT 的 SSH Keys 和 Reinstall 页面只是这一通用流程的一个例子。Serial Console 与 VNC Console 可作为供应商救援工具，但不是正常安装步骤，也不需要提前设置 root 密码。

## 三、设置域名和 DNS

一次只选择一个域名。继续使用 `senyz.top` 最稳妥；如果以后改用 `senyz.art`，就在运行脚本时输入 `senyz.art`，不要让两个域名同时参与同一次迁移。

Cloudflare 中该域名根记录使用：

| 项目 | 值 |
|---|---|
| 类型 | A |
| 名称 | `@` |
| 内容 | 新服务器公网 IPv4 |
| 代理状态 | DNS only（灰色云朵） |
| TTL | Auto 或 300 |
| 同名 AAAA | 不创建；已有则删除 |

如果已经存在 `@` 的 A 记录，直接编辑原记录，不要再添加一条重复记录。只处理这次使用域名的 A/AAAA；不要删除 MX、TXT、邮件或其他无关记录。

不需要 Cloudflare Global API Key、API Token 或 DNS API。脚本使用公开 HTTP Webroot 完成证书签发和自动续期。

可在本地确认 DNS：

Windows：

```powershell
Resolve-DnsName <你的域名> -Type A
Resolve-DnsName <你的域名> -Type AAAA
```

macOS：

```bash
dig +short A <你的域名>
dig +short AAAA <你的域名>
```

A 应只显示新服务器 IPv4，AAAA 应没有结果。DNS 尚未更新时不要运行安装脚本。

## 四、第一次 SSH 登录

同一个 IPv4 重装后，旧主机指纹会变化。只删除这一台服务器的旧记录，不清空整个 `known_hosts`。

Windows：

```powershell
ssh-keygen -R <服务器IPv4>
ssh -i "$HOME\.ssh\vps_senyz_ed25519" root@<服务器IPv4>
```

macOS：

```bash
ssh-keygen -R <服务器IPv4>
ssh -i ~/.ssh/vps_senyz_ed25519 root@<服务器IPv4>
```

首次连接显示主机指纹是正常现象。能从供应商面板查看主机指纹时可进行比对；无法查看时，至少确认正在连接刚刚重装并由自己控制的准确 IPv4，再输入 `yes`。

如果仍使用原来的 Windows 私钥，命令中的密钥路径可以暂时换回：

```powershell
ssh -i "C:\Users\senyz\ssh_passwd\id_rsa.pem" root@<服务器IPv4>
```

## 五、整行复制安装命令

登录 Debian 后，完整复制下面这一行，不需要拆开或理解其中的 GitHub 数字：

```bash
apt-get update && apt-get install -y ca-certificates curl && curl --proto '=https' --tlsv1.2 -fsSLo /root/setup_script.sh https://raw.githubusercontent.com/syubroken/server_proxy_setup/c9910763ec79f279d7c83ec2e1e46b574d43d5f5/setup_script.sh && chmod 700 /root/setup_script.sh && /root/setup_script.sh
```

屏幕只需要你处理三件事：

1. `Domain [senyz.top]`：继续使用 `senyz.top` 就按 Enter；使用其他已完成 DNS 设置的域名就输入它。
2. `Certificate notification email`：输入接收证书通知的邮箱，不会写入 GitHub。
3. `Type YES to build`：确认域名和服务器无误后输入大写 `YES`。

安装通常需要 10 至 30 分钟。软件安装、证书测试和 WARP 验收期间不要主动中断。脚本内部会自动完成下载校验、SSH/UFW、Nginx、证书续期、V2Ray、WARP、路由保护和最终检查。

## 六、怎样判断成功

唯一成功标准是末尾同时出现：

```text
Result: PASS
Clean rebuild complete. Base proxy and WARP both passed verification.
```

随后才会显示 `vmess://...` 导入链接和二维码。客户端链接、UUID 和 WebSocket 路径都属于凭据，不放入 GitHub、普通笔记、聊天或公开截图。

以后需要再次显示时，只执行：

```bash
senyz-show-client
```

再次显示二维码：

```bash
senyz-show-client qr
```

不需要手动寻找 `/root` 文件，也不需要理解“权限 600”；脚本已在后台保护这些文件。

## 七、失败时只做什么

只要没有最终 `Result: PASS`，就把节点视为尚未交付。V2Ray 会保持停止，不会静默使用 VPS 原始出口。

### WARP 注册暂时受限或其他步骤暂停

脚本会进行三次有间隔的受控尝试，不会每几秒无限注册。仍未完成时，等待几小时或隔天重新 SSH 登录，只执行：

```bash
senyz-finish-rebuild
```

它会从已完成的位置继续，不会重装基础系统、重新生成节点或再次询问邮箱。如果提示找不到这条命令，就重新执行第五节的整行安装命令。

### WARP 切换时 SSH 断开

先等待至少 5 分钟再重新 SSH。服务器本机的回退计时器会停止未验证的 WARP 路由，V2Ray 仍保持停止。

如果仍无法 SSH，只在供应商面板执行一次普通重启，等待系统启动后再连接并运行 `senyz-finish-rebuild`。Serial/VNC Console 是可选的最后救援入口，不要求你在里面自行排错；把屏幕内容交给 Codex 即可。

不要因为一次外部限流立即重装，也不要关闭 fail-closed 来换取临时可用。只有继续入口、自动回退和供应商重启都不能恢复时，再评估重装或使用保留的旧流程。

## 八、导入 Shadowrocket

### macOS

1. 复制脚本最后显示的整行 `vmess://...`。
2. 在 Shadowrocket 中从剪贴板导入并选中新节点。
3. 路由选择“代理/Proxy”，开启连接。
4. 先访问普通网站确认网络正常，再使用重要账号。

### iPhone

1. 用 Shadowrocket 扫描终端二维码，或复制链接后从剪贴板导入。
2. 选中新节点，路由选择“代理/Proxy”。
3. 允许 iOS 添加 VPN 配置并开启连接。

macOS 与 iPhone 尽量使用同一节点，不在一个登录会话中频繁切换服务器、国家、WARP 和其他代理。

## 九、以后需要维护什么

正常使用不需要每天登录服务器。每月一次或感觉网络异常时执行：

```bash
senyz-verify-rebuild --require-warp
```

最后显示 `Result: PASS` 即可。它不会打印客户端凭据或私钥。出现 `FAIL` 时保留完整输出交给 Codex，不需要先自行修改配置。

证书由 Certbot Webroot 和 systemd timer 自动续期。Debian 安全更新会自动安装但不会自动重启；只有系统明确需要重启或排错要求重启时，才在供应商面板或 SSH 中安排一次重启并再次验收。

## 十、WARP 能做什么，不能做什么

- 本流程把“WARP 双栈验收成功”作为交付节点的必要条件，且 WARP 后续失效时会停止 V2Ray，减少悄悄退回 VPS 原始出口的机会。
- WARP 可以改变服务器访问目标网站时看到的出口，但它是共享服务，出口 IP 仍可能变化，也不能证明所谓“IP 纯净度”。
- WARP 不能保证任何 AI 服务账号不被限制，不能改变服务地区政策，也不能解决 VPS 入站 IP 被干扰、供应商故障或域名问题。
- 为减少不必要的变量，稳定使用同一服务器、同一节点和同一客户端设置；不要为了追求某个即时检测结果频繁重装或切换线路。

仓库 `legacy/` 中的原始脚本仍保留原内容和原执行行为，只作为新版流程无法完成时的灾难恢复备份。不要再执行不受自己控制、且已停止维护的 `git.io/warp.sh`。
